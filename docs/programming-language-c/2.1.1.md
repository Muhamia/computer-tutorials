### VS和VSC的简单介绍

VS和VSC分别是Visual Studio和VS Code的简称

他们都是在学习过程中经常听到的名字，尤其后者出现的频率似乎更多一点

那么他们到底是什么?能干什么?谁更好?

#### 是什么?
事实上，我认为他们严格意义上不能算是一个东西
##### Visual Studio
VS是一个功能全面的重型集成开发环境(IDE)，它的特点是功能完全而强大，但是占用空间相对来说大一些，我们只开发C语言的话大概会占用2-3个GB
##### VS Code
而VS Code，说难听点就是个大号记事本

当然，倒也没记事本这么低级

VS Code和VS相比，出厂不会自带那么多功能，因此刚出厂的形态确实约等于记事本

但是，这也是VS Code的优点，它足够轻量化，占用空间够小，功能可以自己定制

再加上庞大的插件社会，使得VSC可以扮演多面手的角色

我只需要更改几个插件是否启用，我的VSC就可以从编写C语言的IDE变成markdown(一种电子笔记)文本编辑器

但是，VSC的缺点也很明显，那就是"搞半天还得自己拼"，如果你想用VS Code编写C语言，那么你就得自己手动安装编译器，添加环境变量，同时如果想在运行时弹出exe的控制台窗口，也需要自己改json配置文件

总而言之，我感觉简直麻烦的一批
因此我只拿VS Code做一些工程量小的工作，比如编写这份教程。
这份教程就是我用markdown写的，我现在就在用VS Code编写
除此以外，编程的工作我还是交给VS完成

而且，我认为VS的调试功能十分的舒服，不仅可以逐步调试，还可以设置条件断点(比如我循环进行到第10次再开始逐步调试检查)。简而言之就是找bug很方便

当然，你要是不嫌麻烦，用VSC也没问题
但我还是建议新手从VS开始，如果没条件再用VSC

#### DEV C++是什么?为什么我的老师让我用这个?
DEV C++是一款上古的IDE，功能和VS差不多，但是无论上界面体感还是功能上，我认为都不如更现代化的VS

当然，机考的时候使用VS作为IDE也是没问题的，所以无脑入就行

再说了，你想啊，如果把DEV C++换掉了，那祖传的PPT不又得重新做图片了
这种工作量，哪个老师会去做呀(尬笑)
